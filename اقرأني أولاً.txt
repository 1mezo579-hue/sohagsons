# 📦 دليل نقل نظام ماركت أبناء سوهاج

## الملفات المطلوبة للنقل (انسخ المجلد بالكامل)
```
supermarket-pos-system/
├── src/                    ← كود النظام
├── prisma/                 ← قاعدة البيانات
├── public/                 ← الصور والأيقونات
├── .env                    ← ⚠️ إعدادات سرية (مهم جداً)
├── package.json
├── next.config.mjs
├── ابدأ البرنامج.bat       ← تشغيل يومي
├── تجهيز الجهاز الجديد.bat ← أول مرة فقط
└── إيقاف النظام.bat
```

---

## 🚀 خطوات النقل للجهاز الجديد

### الخطوة 1: متطلبات الجهاز الجديد
- [ ] Windows 10 أو أحدث
- [ ] تثبيت **Node.js LTS** من: https://nodejs.org
- [ ] اتصال إنترنت للإعداد الأول فقط

### الخطوة 2: نسخ الملفات
- انسخ مجلد `supermarket-pos-system` بالكامل على فلاشة
- الصقه في `C:\` على الجهاز الجديد

### الخطوة 3: الإعداد الأول (مرة واحدة فقط)
```
اضغط دبل كليك على: تجهيز الجهاز الجديد.bat
```
انتظر حتى يظهر: **✅ النظام جاهز للاستخدام**

### الخطوة 4: التشغيل اليومي
```
اضغط دبل كليك على: ابدأ البرنامج.bat
```

---

## 🔒 معلومات الأمان والحماية

### كلمات المرور الافتراضية
| المستخدم | كلمة المرور |
|----------|------------|
| admin    | (راجع ملف .env أو اسأل المطور) |

### حماية الكود
- النظام في وضع **الإنتاج** لا يمكن تعديله بسهولة
- الكود مُجمَّع ومضغوط وغير قابل للقراءة
- لا توجد خرائط مصدر (source maps) في نسخة الإنتاج
- قاعدة البيانات على Supabase (سحابية - محمية)

### ⚠️ لا تشارك ملف `.env`
يحتوي على مفاتيح الاتصال بقاعدة البيانات

---

## 🛠️ حل المشاكل الشائعة

### المشكلة: "port 3000 is already in use"
**الحل:** شغّل `إيقاف النظام.bat` ثم أعد التشغيل

### المشكلة: صفحة بيضاء
**الحل:** تأكد من الاتصال بالإنترنت (قاعدة البيانات سحابية)

### المشكلة: "Cannot find module"
**الحل:** شغّل `تجهيز الجهاز الجديد.bat` مجدداً
